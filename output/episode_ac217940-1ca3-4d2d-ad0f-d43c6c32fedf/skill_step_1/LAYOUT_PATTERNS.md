# Layout Patterns — Mandatory Slide Structure

## Slide Anatomy (Top to Bottom)

Every slide MUST follow this vertical structure:

```
┌─────────────────────────────────────────────────┐
│ TOP MARGIN (0.5")                               │
│                                                 │
│ TITLE ZONE (y: 0.5" to ~1.1")                  │
│   - Insight-driven title, left-aligned          │
│   - Georgia/serif, 24-30pt bold, navy #192A3E   │
│                                                 │
│ DIVIDER LINE (y: ~1.15")                        │
│   - Horizontal rule spanning left to right margin│
│   - Color: #8C9DAD, weight: 0.75-1.0pt         │
│                                                 │
│ CONTENT ZONE (y: 1.3" to ~6.5")                │
│   - Tables, charts, text blocks                 │
│   - See content patterns below                  │
│                                                 │
│ SOURCE LINE (y: ~6.6")                          │
│   - "Source: ..." in Calibri 8pt, #5A6A7A       │
│                                                 │
│ FOOTER DIVIDER (y: ~6.85")                      │
│   - Thin line, #8C9DAD, 0.5pt                   │
│                                                 │
│ FOOTER (y: ~6.95" to 7.0")                      │
│   - Left: "McKinsey & Company"                  │
│   - Right: Page number                          │
│   - Calibri 8pt, #5A6A7A                        │
│                                                 │
│ BOTTOM MARGIN (0.5")                            │
└─────────────────────────────────────────────────┘
```

## Slide Dimensions
- Standard: 13.333" × 7.5" (widescreen 16:9)
- All positioning assumes this dimension

## Content Zone Patterns

### Pattern A: Data Table (PREFERRED for most analytical slides)
- Use when presenting metrics, comparisons, projections
- Table positioned centered in content zone
- Column widths proportional to content
- Header row: navy bg (`192A3E`), white text, bold
- Data rows: alternating white / light blue-grey (`D6E4F0`)
- Borders: thin lines in `8C9DAD`
- Numbers right-aligned, text left-aligned

### Pattern B: Two-Column Layout
- Left column (60%): primary content (table or key findings)
- Right column (40%): supporting detail or callout box
- Vertical gap: 0.3" between columns

### Pattern C: Key Metrics + Detail
- Top row: 3-4 key metric boxes (navy text, light blue-grey background)
- Below: supporting table or bullet points
- Metric boxes: large number (24pt bold), label below (11pt)

## Mandatory Structural Checklist
Before finalizing any slide, verify:
- [ ] White background (`FFFFFF`)
- [ ] Insight-driven title (not a topic label)
- [ ] Horizontal divider line under the title
- [ ] Structured data presentation (table preferred)
- [ ] Source line if external data is referenced
- [ ] Footer with company name and page number
- [ ] Footer divider line
- [ ] All text is legible (high contrast against background)
- [ ] Only navy/grey/blue-grey palette used
- [ ] Professional, restrained aesthetic — no decorative elements
